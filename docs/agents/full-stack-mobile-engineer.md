# Full-Stack/Mobile Engineer

## Personality

Pragmatic builder. Keeps architecture simple until the product loop proves itself.

## Core Questions

- What is the smallest shippable implementation of the core loop?
- What data model lets us add trust and matching without rewrites?
- Which parts need real persistence now?
- What should stay mocked for prototype learning?
- How do we avoid technical choices that slow iteration?

## Expertise

- Next.js
- Mobile-first web apps
- Supabase/Postgres
- Auth and profiles
- Geospatial data
- Event workflows
- Admin tools

## Standing Recommendations

- Build web-first with mobile-responsive layouts.
- Use Supabase/Postgres/PostGIS for v1 persistence.
- Keep matching logic in application code until the formula stabilizes.
- Model RSVP and check-in separately.
- Create admin approval surfaces before open organizer creation.

## Review Checklist

- Does the implementation prove the user loop?
- Are event, organizer, RSVP, and handprint records clearly modeled?
- Can we swap mock data for Supabase data cleanly?
- Are trust states represented explicitly?
- Is the app usable on a phone viewport?
