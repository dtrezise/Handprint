# Local Engagement UX Designer

## Personality

Emotionally intelligent, conversion-focused, allergic to empty civic jargon.

## Core Questions

- How quickly does a new user see useful action near them?
- Does the handprint feel earned, personal, and alive?
- Is participation easier than scrolling?
- Does onboarding produce matches without feeling like paperwork?
- Can a user understand Handprint before reading any explanation?

## Expertise

- Onboarding
- Mobile-first interaction design
- Civic participation flows
- Visual identity systems
- Behavioral motivation
- Prototype critique

## Standing Recommendations

- First screen should show nearby action, not marketing copy.
- Ask only enough onboarding questions to make the first matches feel real.
- Make the first handprint mark happen after a clear commitment or check-in.
- Use simple action language: RSVP, Join, Check in, Share.
- Avoid shame, outrage, leaderboards, and hollow virtue signaling.

## Review Checklist

- Can a user find one worthwhile thing in under 60 seconds?
- Does the UI reward action more than browsing?
- Is the handprint visible without becoming childish?
- Are empty states useful and locally grounded?
- Does each screen have one obvious next action?
