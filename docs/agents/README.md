# Handprint Agentic Operating System

This folder defines the specialist roles Handprint should use throughout product development. Treat these as reusable lenses for planning, review, implementation, and red-team work.

## How To Use The Roles

For each major decision, route the work through the smallest relevant set of agents:

1. Product scope or launch strategy: Founder/Product Strategist + Community Operations Lead.
2. Matching, feeds, or personalization: Sphere Algorithm Designer + Civic Trust & Neutrality Architect.
3. User flows, onboarding, handprint visualization: Local Engagement UX Designer + Data & Impact Systems Agent.
4. Event creation, verification, check-in, moderation: Full-Stack/Mobile Engineer + Civic Trust & Neutrality Architect.
5. Political, privacy, organizer, or child/school use risks: Legal/Policy Risk Advisor before implementation.

## Operating Cadence

- Weekly product review: Product Strategist, UX Designer, Engineer.
- Weekly trust review: Trust Architect, Legal Advisor, Data Agent.
- Launch readiness review: all roles.
- Post-pilot review: Community Ops, Data Agent, Product Strategist.

## Shared Product Doctrine

- Handprint monetizes participation, not attention.
- The handprint visualization is core identity infrastructure, not decoration.
- Neutrality must be operationally legible, not merely asserted.
- Cold start is the first existential risk.
- Launch density matters more than feature breadth.
- Verified local usefulness beats global ambition in v1.
