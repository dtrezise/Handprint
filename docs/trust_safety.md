# Trust, Safety, And Neutrality Foundation

## Neutrality Principle

Handprint does not endorse user ideology. It matches people to local actions based on user-selected interests, location, availability, skill fit, World Enabler trust, and event quality.

Neutrality must be implemented through transparent rules, review processes, and explainable ranking.

## V1 Trust Model

V1 should use controlled supply:

- Verified World Enablers only.
- Events reviewed before public listing.
- Sensitive categories require extra review.
- Users can report events.
- Admins can suspend World Enablers.

Trust, World Enabler accolades, reward eligibility, and Handprint points must never be sold.

## World Enabler Verification

Minimum fields:

- Legal or public organization name when applicable.
- Contact name.
- Email and phone.
- Website or social proof.
- Community affiliation.
- Event history or reference.
- Agreement to World Enabler terms.

## Event Review Criteria

Approve events that are:

- Locally actionable.
- Truthfully described.
- Safe enough for public listing.
- Clear about location, time, expectations, and World Enabler.
- Within launch community scope.

Reject or escalate events involving:

- Harassment, intimidation, or doxxing.
- Violence or illegal activity.
- Misleading fundraising.
- Candidate, campaign, or election activity before legal review.
- Medical, legal, or financial claims.
- Minors or school groups without additional safeguards.

## Sponsored And Fundraising Controls

Sponsored visibility is allowed only when it is plainly labeled and separated from reward eligibility.

Rules:

- Sponsored placement does not create Handprint points.
- Sponsored placement does not create World Enabler trust tier, verification, or accolades.
- Users can mark sponsored or awareness content as not interested.
- Targeting uses declared interests, location, and event category, not sensitive inferred attributes.
- Fundraising listings must state beneficiary, intended use, and post-campaign impact receipt plan.

Financial transparency can become an optional add-on later. V1 should keep the visible app focused on what the event accomplished and who affirmed it.

## Affirming Interaction Standard

Handprint should allow World Changers and World Enablers to talk, invite, thank, recruit, and follow up, but the culture cannot become open-season comment behavior.

Every public post, share caption, comment, and recruitment message should pass through an Affirmation Agent before publishing.

The agent should:

- Preserve the user's core sentiment when possible.
- Flag personal attacks, mocking, pile-ons, dehumanizing language, and broad blame.
- Flag unsupported superlative or verification claims.
- Suggest a rewrite that turns concern into useful action, gratitude, invitation, or repair.
- Prefer rewrite-first coaching over punishment.
- Escalate repeated or severe abuse to human review.

The product standard is not "only say happy things." Users can express frustration, urgency, concern, grief, and disagreement. The required turn is that the message must point toward doing something useful, affirming someone, or inviting better action.

For V1, this can begin as a visible prototype composer and rule set. Before public scale, it should become a model-backed moderation service with audit logs, appeal states, reviewer tools, and privacy rules for sensitive messages.

## Ranking Transparency

Each recommendation should include match reasons, such as:

- Near you
- Matches your interest in food security
- Uses your logistics skill
- This weekend
- Hosted by a verified World Enabler

## Data Minimization

V1 should avoid collecting:

- Sensitive demographic attributes
- Political party affiliation
- Precise location history
- Contact graph imports
- Unneeded identity documents

## Incident Response

For pilot:

1. Remove or hide the event if immediate harm is plausible.
2. Preserve audit notes.
3. Contact World Enabler when appropriate.
4. Notify affected users if event details materially change.
5. Review policy gap before reinstating similar events.
