# Handprint V1 Narrative Spec

Last updated: July 10, 2026

## Core Story

Handprint helps a person find one useful action within reach, show up, get that contribution affirmed, and share the proof in a way that invites someone else to join.

The V1 story is:

1. Find one useful action.
2. Understand what you will actually do.
3. Show up.
4. Get confirmed by a trusted World Enabler.
5. Watch your Handprint grow.
6. Wave that Handprint outward to invite one more person.

## Four Primary Places

The member experience has four primary destinations. New functions must belong to one of these before a new top-level destination is considered.

### Reach

Find useful opportunities within the physical and practical reach of the user's hands. Reach owns search, location, filters, event discovery, event detail, RSVP, check-in, and routes into World Enabler pages.

### Print

See the Handprint being left behind. Print owns World Changer rank, verified marks, badges, goals, points, rewards, credentials, participation history, and the pathway to becoming a World Enabler. World Enabler identity and tools are a role within Print, not a competing public destination.

### Wave

Send a Handprint outward. Wave owns the public Handprint, QR and link sharing, milestone stories, invitations, praise messages, affirmation credits, and the Affirmation Agent that protects constructive interaction.

### Shake

Connect while people are physically together. Shake begins by listening for a nearby in-person connection, then becomes the useful network created from those meetings: World Changers, World Enablers, and their next actions.

Hi-Five is not a destination. It is a lightweight animated reaction to a badge or achievement. It celebrates proof of useful action without creating another feed or page.

Profile and settings are utilities accessed from the profile icon. Trust review and development controls are role-gated utilities, not part of the everyday member navigation.

## First Five Minutes

The first five minutes should not ask users to understand the whole ecosystem. It should help them believe:

- There is something useful near me.
- I can do it without already being an expert.
- This is about real-world action, not online performance.
- My contribution can become visible proof.
- I can invite someone else without bragging in an empty way.

## Screen Rule

Every screen should answer three questions:

- What can I do next?
- Why does it matter?
- How is this verified or made trustworthy?

If a screen cannot answer those questions, it belongs behind an advanced, reviewer, or future-portal layer.

## Reach Rule

Reach should lead with one useful action, not a dashboard of systems. Search and filters are tools, but the emotional hook is the action story:

- Why it matters.
- What you will actually do.
- Who benefits.
- What happens after you show up.

## Reward Rule

Rewards expand reach. They should never feel like prizes replacing the value of the action.

Badges are the primary dopamine unit for V1. Points, credits, scholarships, sponsor rewards, and real-world rewards should support the verified-action story, not compete with it.

## Trust Rule

Every official claim needs a visible verifier:

- Official Handprint means the event or contribution is verified.
- Unofficial example means it is a useful lead, not yet a Handprint partner.
- Awareness only means it can matter, but does not earn rewards unless connected to a verified action.
- Sponsored discovery must never imply bought trust or bought impact.

## World Enabler Rule

World Enablers should feel like stewards, not advertisers. Their V1 promise is:

- Create useful action.
- Recruit the right people.
- Confirm what happened.
- Publish impact receipts.
- Build proof that can help with trust, grants, recruitment, and community credibility.

## Wave Rule

Wave should start simple:

- One QR.
- One public Handprint link.
- One invitation message.
- One share preview.

Advanced share kits, moderation queues, social history, and platform-specific exports should remain secondary until the public Handprint loop is unmistakable.

## Shake Rule

Shake should help users make an in-person connection first, then find more useful action through those trusted people and World Enablers. It should not become a generic social feed before the action loop is proven.

V1 Shake should prioritize:

- Nearby listening while the page is open.
- A deliberate physical shake gesture and clear connection confirmation.
- People and World Enablers met or deliberately added.
- Their next useful actions.
- Their public Handprint pages.
- Recruiting paths into verified events.
