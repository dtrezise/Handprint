# Launch Plan

## Recommended Pilot

Launch Handprint in one dense, trust-rich community before opening broadly.

Best first contexts:

1. Campus/community college ecosystem.
2. Faith and nonprofit coalition in one city district.
3. Employer civic engagement program.
4. Neighborhood association network.

Default recommendation: campus/city hybrid pilot because it can produce recurring events, social spread, and a concentrated user base.

## Pre-Launch Targets

- 10-20 verified World Enablers.
- 50-100 seeded events or actions.
- 5-10 event categories.
- One named pilot geography.
- Manual support channel.
- Admin approval workflow.

## First 10 Event Categories

1. Food support
2. Cleanup and beautification
3. Mentoring and tutoring
4. Mutual aid
5. Faith/community service
6. Civic forums
7. Fundraisers
8. Health and wellness support
9. Neighborhood safety and preparedness
10. Arts and cultural community building

## World Enabler Verification Rules

- Verify identity or affiliation before publishing events.
- Require clear event purpose, location, time, and expectations.
- Require World Enabler contact information.
- Require terms acceptance.
- Manually review first three events from every World Enabler.
- Escalate political, fundraising, youth, medical, or high-risk events.

## First 30 Days

Week 1:

- Confirm pilot geography.
- Recruit anchor World Enablers.
- Seed event catalog.
- Run internal prototype testing.

Week 2:

- Invite first user cohort.
- Measure match-to-RSVP.
- Interview users after RSVP.
- Fix onboarding and event clarity.

Week 3:

- Add World Enabler feedback loop.
- Improve handprint marks.
- Expand event supply.
- Start referral/share tests.

Week 4:

- Review retention and attendance.
- Decide whether to deepen the pilot or change wedge.
- Prepare investor/advisor demo if metrics are promising.
