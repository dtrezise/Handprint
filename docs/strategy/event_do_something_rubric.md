# Event "Do Something" Rubric

## Purpose

Define how Handprint decides whether an event earns Handprint rewards. The goal is to protect the product from becoming a polarized activism feed while still allowing users to discover awareness events, advocacy events, and causes they care about.

## Core Principle

Handprint rewards verified doing, not merely signaling.

An event can be discoverable in Handprint without being reward-eligible. Rewards should be reserved for actions that create a concrete, verifiable benefit for people, places, organizations, or community capacity.

## Strategic Assessment

This is a foundational product-governance idea. Without a clear "do something" standard, Handprint can drift into the same attention, outrage, and ideological performance loops it is meant to counter.

The challenge is that awareness can matter. Protests, rallies, issue campaigns, and public demonstrations may be meaningful to users. But if Handprint grants World Changer points for ideological attendance alone, the app risks becoming a status system for political identity rather than contribution.

The solution is to separate:

1. **Discoverable events:** events users may want to know about.
2. **Promoted/advertised events:** paid or sponsored visibility, with targeting and user controls.
3. **Reward-eligible Handprint events:** verified actions that do concrete good.

## Recommended Expertise Roles

### Civic Neutrality Lead

Owns the policy distinction between cause discovery and Handprint reward eligibility.

### Impact Measurement Lead

Defines what counts as concrete benefit and what evidence is needed.

### Trust And Safety Lead

Reviews polarizing, misleading, unsafe, or contested events.

### World Enabler Verification Lead

Confirms whether the World Enabler and beneficiary are legitimate and whether claimed impact can be verified.

### Advertising Ethics Lead

Designs paid promotion rules so revenue does not override user trust.

### Recommendation And Controls Designer

Builds user filters, "not interested" controls, sensitive-topic preferences, and transparent ranking explanations.

### Legal And Policy Advisor

Reviews political, campaign, fundraising, youth, medical, financial, and regulated categories before launch.

## Event Classes

### Class A: Reward-Eligible "Do Something" Events

These events create concrete benefit and can earn Handprint points, badges, and World Changer progress.

Examples:

- Food sorting or distribution.
- Neighborhood cleanup.
- Tutoring or mentoring with safeguards.
- Fundraiser where funds go to a verified community benefit.
- First aid training.
- Tool library intake.
- Mutual aid intake support.
- Disaster preparedness kit assembly.
- Civic forum that produces an actionable community output, such as a published priorities report, approved volunteer plan, or service signup pipeline.

### Class B: Discoverable Awareness Events

These events may matter to users, but do not earn Handprint rewards by default because the immediate output is awareness, expression, solidarity, or persuasion.

Examples:

- Protest.
- March.
- Rally.
- Awareness gathering.
- Issue campaign meeting.
- Public demonstration.
- Petition-signing table without a concrete service output.

These may be listed if they pass safety, legality, World Enabler verification, and user-interest controls. Participation should not earn World Changer points unless the event includes a separate verified action component.

Founder decision: include awareness/protest-style listings in V1, but keep them clearly labeled and reward-ineligible unless they add verified action components.

### Class C: Sponsored Cause Promotion

These are paid or promoted placements for causes, events, or organizations.

They must be clearly labeled as sponsored or promoted. They should be targeted only through user-declared interests, not sensitive inferred attributes. Users must be able to hide, block, or mark "not interested."

Sponsored visibility must not create Handprint rewards by itself.

Founder decision: include sponsored cause/event promotion in V1, with strict labeling, user controls, and no rewards for sponsored visibility alone.

### Class D: Not Allowed Or Escalated

Events should be rejected or escalated if they involve:

- Harassment, intimidation, or doxxing.
- Violence or illegal activity.
- Misleading fundraising.
- Candidate, campaign, or election activity before legal review.
- Medical, legal, or financial claims without safeguards.
- Youth or school groups without additional review.
- Vague "raise awareness" claims presented as measurable impact.
- Ideological targeting based on sensitive inferred traits.

## Reward Eligibility Rubric

An event must pass all required criteria to earn Handprint rewards.

### 1. Concrete Action

The event asks participants to do a specific thing beyond showing up.

Pass examples:

- Pack 80 pantry boxes.
- Clean 3 blocks.
- Staff an intake table.
- Assemble 60 storm kits.
- Raise funds for a verified local shelter repair.

Fail examples:

- Attend a rally.
- Hold a sign.
- Listen to speeches.
- Increase awareness.

### 2. Beneficiary Or Community Benefit

The event has a clearly named beneficiary, community asset, or capacity outcome.

For fundraisers, the destination and intended use of funds must be clear. "Youth group trip to Disneyland" is not automatically a Handprint-reward event unless the trip itself has a verified community-service purpose. A bake sale for a food pantry, shelter, school supply drive, local medical fund, or community repair fund can qualify if the beneficiary and use are verified.

### 3. Verifiable Output

The event can produce evidence after completion.

Examples:

- Attendance confirmed by World Enabler.
- Items packed, funds raised, kits assembled, rooms cleaned, forms completed, hours tutored, calls made, meals delivered.
- Beneficiary attestation.
- Photos or receipts where appropriate.
- Post-event World Enabler confirmation.

### 4. Non-Harm And Safety

The event must be safe enough for public listing and must not create foreseeable harm, coercion, harassment, or illegal activity.

### 5. Neutral Category Framing

The event must be described in terms of action and beneficiary, not ideological superiority.

Prefer:

- "Help distribute food to 80 households."
- "Join a moderated forum to collect park funding priorities."

Avoid:

- "Fight evil people."
- "Prove you care."
- "Defeat the other side."

### 6. World Enabler Accountability

The World Enabler must be verified enough for the event's risk level and must agree to post-event confirmation rules.

## Awareness-To-Action Bridge

Awareness events can become reward-eligible if they include a verified action component.

Examples:

- Rally plus food drive: reward only the verified food-drive work.
- March plus voter registration drive: escalated/legal review; reward only compliant registration support if allowed.
- Awareness event plus blood drive: reward the verified blood-drive support.
- Public meeting plus volunteer signup and follow-up shift: reward the completed follow-up shift, not just attendance.
- Issue talk plus community cleanup after: reward the cleanup.

This lets Handprint support civic interest without rewarding performative attendance.

Handprint should actively encourage awareness organizations, activist groups, corporations, and other public-interest groups to create their own Handprint through do-ism opportunities. Examples include post-event cleanup, service booths, action signup tables, legal-aid or documentation support after legal review, volunteer shifts, donation drives with verified beneficiaries, and employee/community service events. The product posture is not "activism is unwelcome"; it is "awareness becomes more powerful when it creates verified help."

## Advice On Activism Versus "Do-Ism"

Handprint should not ban activism from discovery, but it should not treat activism attendance as equivalent to concrete service.

Recommended policy:

- Awareness/protest/advocacy events may be discoverable.
- They do not earn Handprint points by default.
- They can be sponsored only with clear labeling and user controls.
- They can earn rewards only for verified action components.
- Users can filter them out.
- Handprint should not infer political interests from sensitive data.
- Handprint should not sell polarizing outrage as engagement fuel.

This protects the app's identity: Handprint is a local civic action engine, not a social network for ideological mobilization.

## Advertising And Promotion Rules

Paid or promoted awareness events can exist, but require strict boundaries:

- Clearly label sponsored placements.
- Let users hide the sponsor or topic.
- Do not grant rewards for ad participation.
- Do not use manipulative urgency or shame copy.
- Do not target based on sensitive inferred political identity.
- Keep ranking explanations transparent.
- Allow reports for misleading or polarizing content.

Revenue should never buy World Changer points.

## Recommended Product Model

Add event eligibility fields:

- `listingType`: `action`, `awareness`, `sponsored`, `training`, `fundraiser`
- `rewardEligible`: boolean
- `rewardReason`: short explanation
- `impactClaim`: concrete expected output
- `beneficiary`: named recipient/community benefit
- `verificationPlan`: how completion will be confirmed
- `sensitiveReview`: boolean
- `sponsorDisclosure`: optional paid placement label

## Recommended UX Copy

For reward-eligible events:

> Earns Handprint rewards because this event has a verified action, beneficiary, and completion check.

For awareness events:

> Discoverable cause event. No Handprint rewards unless you complete a verified action connected to it.

For fundraisers:

> Reward eligibility depends on verified beneficiary and use of funds.

## Roadmap

### Phase 1: Policy And Data Model

- Define listing classes.
- Define reward eligibility fields.
- Add review criteria to World Enabler submission.
- Add public explanation text.

### Phase 2: Prototype

- Add one reward-eligible fundraiser.
- Add one awareness-only event.
- Add UI labels: "Earns rewards" and "Awareness only."
- Hide reward points on awareness-only events.
- Add "not interested" control for awareness/sponsored events.

### Phase 3: Review Workflow

- Add admin review fields for concrete action, beneficiary, verification plan, and sensitive review.
- Add World Enabler post-event confirmation for impact outputs.
- Add beneficiary attestation where needed.

### Phase 4: Sponsored Discovery

- Add sponsored event labeling.
- Add user preference controls.
- Add ad quality review.
- Keep sponsored events separate from rewards.

### Phase 5: Measurement And Governance

- Track whether reward-eligible events produce verified outputs.
- Track user trust in event labels.
- Audit point allocation by category.
- Review polarizing categories before expanding.

## Recommended V1 Decision

Adopt the "Do Something" standard now:

- Only concrete, verified action earns Handprint rewards.
- Awareness/protest/advocacy events may be discoverable but do not earn rewards by default.
- Fundraisers qualify only when beneficiary and use of funds are verified.
- Sponsored visibility is allowed only with clear labeling, user controls, and no rewards.
- Handprint points are never for sale.

## Founder Decisions

- Awareness/protest-style events are included in V1 discovery.
- Paid sponsored cause/event promotion is included in V1.
- Sponsored placement and awareness attendance never earn rewards by themselves.
- Handprint should help awareness groups, activist organizations, and corporations create concrete do-ism extensions that can earn rewards.
